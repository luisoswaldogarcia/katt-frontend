# Actualizar documentación

Cuando el usuario te pida "actualiza docs" o "update second-brain":

1. Explora src/ para identificar la estructura actual de módulos
2. Lee los archivos existentes en ~/second-brain/
3. Actualiza ~/second-brain/README.md y cada archivo .md de módulo
4. Mantén el formato: heading, [← Mapa](README.md), Deps, Usado por, descripción, rutas, archivos clave
5. NO frontmatter, NO mermaid, NO código copiado
